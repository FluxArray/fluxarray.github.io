## C
We could solve this greedily using a **prefix sum** for array **A** (as we get all the previous point too) and **prefix max** for array **B** (as we could just spend the remaining k here on this maximum value without going further in **A**) and just choose the best one from all the possible outcomes.
:::spoiler code
```cpp
void solve() {
    int n, k; cin >> n >> k;
    vi a(n), b(n);
    for (int i = 0; i < n; i++) { cin >> a[i]; }
    for (int i = 0; i < n; i++) { cin >> b[i]; }
 
    int pref = 0, maxb = 0, ans = 0;
    for (int i = 0; i < n; i++) {
        pref += a[i];
        ckmax(maxb, b[i]);
        if (k > i) {
            ckmax(ans, maxb * (abs(k - i - 1)) + pref);
        }
    }
 
    cout << ans << '\n';
}
```

## D
For this problem what I did was take the indexes array **sort the indexes itself** in decreasing order of the original array and now we check the first **min(3, n)** of the arrays using 3 loops such that the indexes shouldn't the equal and just update the ans with the maximum for every possible permutation, cause the maximum can't be outside of these 3 numbers.
:::spoiler code
```cpp
void solve() {
    int n; cin >> n;
    vi a(n), b(n), c(n);
    for (int i = 0; i < n; i++) { cin >> a[i]; }
    for (int i = 0; i < n; i++) { cin >> b[i]; }
    for (int i = 0; i < n; i++) { cin >> c[i]; }
 
    auto itr0 = [&](int i, int j) { return a[i] > a[j]; };
    auto itr1 = [&](int i, int j) { return b[i] > b[j]; };
    auto itr2 = [&](int i, int j) { return c[i] > c[j]; };
 
    vi ia(n), ib(n), ic(n);
    iota(ALL(ia), 0); iota(ALL(ib), 0); iota(ALL(ic), 0);
 
    sort(ALL(ia), itr0);
    sort(ALL(ib), itr1);
    sort(ALL(ic), itr2);
 
    int ans = -1;
    for (int i = 0; i < min(3, n); i++) {
        for (int j = 0; j < min(3, n); j++) {
            for (int k = 0; k < min(3, n); k++) {
                int x = ia[i];
                int y = ib[j];
                int z = ic[k];
                if (x != y && y != z && x != z) {
                    ckmax(ans, a[x] + b[y] + c[z]);
                }
            }
        }
    }
 
    cout << ans << '\n';
}
```

## E1/E2
This was a nice problem, so if you look at the **score** which is **(A - B)** so **A** wants to maximise this while **B** wants to minimise according to the question each time a player chooses and index let's say **A** the **score** increases by **(ai - 1)** while when **B** chooses and index the **score** decreases by **(bi - 1)** so at each index either the score can increase by ai - 1 or decrease by bi - 1 and given that each player is playing optimally, 
Alice would want to decrease Bob's contribution to score as well as keep maximising her score and Bob would want to decrease Alice's contribution to score as well as keep maximising his score.
So we want a way to control **both** (maximising own score and minimising opponents) of these at the same time, a way to do this is called **opportunity cost**(kind of like in economics :o) but I call it **sum sorting** which is as the name suggests sorting by their sum so if we reverse sort by their sum the elements gets sorted by decreasing value of their "value" so each turn Alice or Bob will basically choose the current index and gain their respective contribution to score this is what an optimal play would be like.
Btw this value that is talked here is the relative difference between the 2 contribution so like the difference is 
**ai + bi - 2** so you can see why we used sum sorting.
:::spoiler code
```cpp
void solve() {
    int n; cin >> n;
    vector<pl> ab(n);
    for (int i = 0; i < n; i++) { cin >> ab[i].fi; }
    for (int i = 0; i < n; i++) { cin >> ab[i].se; }
 
    auto itr =  [](const auto x, const auto y) { return x.fi + x.se > y.fi + y.se; };
    sort(ALL(ab), itr);
 
    ll ans = 0, turn = 1;
    for (int i = 0; i < n; i++) {
        if (turn & 1) {
            ans += ab[i].fi - 1;
        } else {
            ans -= ab[i].se - 1;
        }
        turn ^= 1;
    }
 
    cout << ans << '\n';
}
```

## F
This was also a nice problem, so basically for the solution let's look at an equivalent question which is something like this we have 100 balls 90 red and 10 blue so how would you make the pair of course we can make 10 pairs using the 10 blue balls and 10 from the red now let's say there were 50 red and 50 blue then you can make all of them into pairs 
let's increase the type of balls let's say now there are 150 balls 50 red, blue, green we can also make all the balls into pairs, so if you notice there can only be 2 type of cases either one ball is greater than half the total or all of them are smaller than half so more formally

let total be n and maximum of all the balls be mx
**condition 1:**
$mx > n - mx$ in this case there can be a total maximum of (n - mx) / 2 pairs possible.
**condition 2:**
$mx <= n - mx$ in this case there can be a total of (n) / 2 pairs possible.
so this problem can be translated to the original problem since we can't pair the nodes of the single **branch** with each other so all of them can be considered a single color like red and the other branches different colors and the **mx** now becomes whatever the heaviest child of them each for each layer/iteration so the heavy child can give his nodes for the nodes outside of the branch for pairing.
So if you notice there is like a recursion like process happening each iteration, we look at the heavy child and check the 2 conditions (ie. condition 1 and the base case is the condition 2) again and again till we reach the base case.

So each iteration we would carry(c) the outside nodes which we had made pair with the heavy child 
$mx - c <= rest$ (base case)
$mx - c > rest$ 
where **rest** is the nodes outside this child in the current iteration.
All of this could be done using **dfs** 
:::spoiler code
```cpp
void dfs1(vi& sz, vector<vi>& adj, int u) {
    sz[u] = 1;
    for (auto v : adj[u]) {
        dfs1(sz, adj, v);
        sz[u] += sz[v];
    }
}
 
int dfs2(vi& sz, vector<vi>& adj, int u, int k) {
    int tot = sz[u] - 1;
    if (tot == 0) { return 0; }
    int mx = -1, hc = -1;
 
    for (auto v : adj[u]) {
        if (ckmax(mx, sz[v])) { hc = v; }
    }
 
    int rest = tot - mx;
    if (mx - k <= rest) {
        return (tot - k) / 2;
    } else {
        return rest + dfs2(sz, adj, hc, max(0, rest + k - 1));
    }
}
 
void solve() {
    int n; cin >> n;
    vv(int, adj, n + 1);
    for (int i = 2; i <= n; i++) {
        int p; cin >> p;
        adj[p].pb(i);
    }
 
    vi sz(n + 1);
    dfs1(sz, adj, 1);
    cout << dfs2(sz, adj, 1, 0) << '\n';
}
```

## G1/G2
In this question one could find the first part easily by look at some examples and noticing that only those segments which don't overlap should be counted, as if it overlaps then we can just turn on the bulb which is in the overlapping region and use that bulb to turn on it's other pair so for example
1 2 1 2 we could turn on 2 using the 2nd condition and then turn on the last 2
and another example is
1 2 2 1 3 3 so here there are 2 separate segments 1 2 2 1 and 3 3.
So we can use prefix xor for solving this part whenever we reach 0 we can just count++.
One thing with this method is that it can give false positives such that 1 2 3 is 0 but isn't a segment so to prevent this we can use a random number generator and use those as a weight for this prefix.

The second part is the trickiest so for each of our segment we will only choose every element except those which is like a segment inside a segment so for example
1 2 1 2 we can choose any of these 4 
while for 1 2 2 1 we can only chose 1st and 4th 
Using the prefix xor for a segment like $ABBA$ the prefix xor array would be $A, A \oplus B, A, 0$.
So a thing to notice here is that a segment inside the segment becomes 0 first after which the main segment becomes 0
so for this we could use a hashing technique to not count the segments inside a segment, in the hash we would store the end index of the element of xor prefix so like for $A$ it's 3 and for $A \oplus B$ it's 2 and these are the only elements here and when we iterate over a segment we would check the condition if the last occurrence of this pref is greater that the current position if so then we bingo we found a sub-segment and we would now remove this from the main segment that's it for this problem
:::spoiler code
```cpp
void solve() {
    int n; cin >> n;
    vi c(2 * n + 1);
    for (int i = 1; i <= 2 * n; i++) { cin >> c[i]; }
 
    vl weight(n + 1);
    for (int i = 1; i <= n; i++) { weight[i] = rng64(); }
 
    vl pref(2 * n + 1);
    for (int i = 1; i <= 2 * n; i++) {
        pref[i] = pref[i - 1] ^ weight[c[i]];
    }
 
    unordered_map<ll, int> last;
    for (int i = 2 * n; i >= 0; i--) {
        if (last.find(pref[i]) == last.end()) { last[pref[i]] = i; }
    }
 
    int count = 0;
    ll ways = 1;
    int strt = 1;
    for (int ed = 1; ed <= 2 * n; ed++) {
        if (pref[ed] == 0) {
            count++;
            int temp = strt;
            int len = 0;
            while (temp < ed) {
                int nxt = last[pref[temp]];
                if (nxt > temp) {
                    len += nxt - temp;
                    temp = nxt;
                } else {
                    temp++;
                }
            }
            int temp1 = (ed - strt + 1) - len;
            ways = (ways * temp1) % MODD;
            strt = ed + 1;
        }
    }
    cout << count << ' ' << ways << '\n';
}
```
